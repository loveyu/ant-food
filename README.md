# 蚂蚁找食物 - 蚁群算法模拟

一个基于 Canvas 的蚁群算法（Ant Colony Optimization）交互式模拟程序。

## 在线演示

直接在浏览器中打开 `index.html` 即可运行。

## 操作说明

1. 点击画布设置巢穴位置（首次点击设置位置，拖拽调整大小）
2. 使用"画食物"按钮在画布上添加食物源
3. 使用"画水"按钮在画布上添加水源
4. 点击"开始"运行模拟，"暂停"停止，"重置"重新开始

## 算法说明

### 基本原理

模拟使用三种信息素（食物信息素、巢穴信息素、水源信息素）引导蚂蚁行为。蚂蚁通过以下机制协作：

- **外出蚂蚁**沿食物信息素梯度找到食物源
- **回程蚂蚁**直线返回巢穴，沿途沉积食物信息素形成路径
- **巢穴信息素**由外出蚂蚁沉积，形成从巢穴向外递减的梯度

### 状态机

蚂蚁在不同状态间切换：

| 类型   | 颜色     | 行为                     |
| ------ | -------- | ------------------------ |
| 0 深灰 | 觅食     | 跟随食物信息素或随机搜索 |
| 1 红色 | 找水     | 跟随水源信息素           |
| 4 紫色 | 走向食物 | 沿固定路径走向食物       |
| 5 绿色 | 原路返回 | 携带食物直线返回巢穴     |
| 7 蓝色 | 固定回巢 | 沿固定路径返回巢穴       |
| 8 黄色 | 水路径   | 沿水源信息素路径移动     |

### 信息素跟随策略

外出蚂蚁采用**局部梯度上升**策略：优先检查8个相邻像素的信息素浓度，选择最强方向移动；若相邻像素无信息素，再扩大到视野范围搜索。

## 收敛性分析

### 仅有食物时

当场景中只有巢穴和食物时，蚁群算法容易收敛：

- 蚂蚁随机搜索找到食物后返回，沿途留下食物信息素
- 后续蚂蚁沿信息素梯度快速找到食物，形成正反馈
- 路径迅速从随机探索收敛到稳定的最优路径

### 引入水源后

当水源作为第三个要素加入后，问题复杂度显著提升：

1. **信息素维度冲突**：三种信息素（食物、巢穴、水源）在空间上相互干扰，蚂蚁需要在多个梯度间做出选择
2. **状态切换开销**：蚂蚁需要在觅食、找水、回巢三种目标间切换，频繁的状态转换打断路径形成
3. **纯信息素方案的局限**：仅靠信息素浓度单一维度，无法同时编码"去哪里"和"做什么"两个决策，难以处理多目标协同

### 当前实现的折中方案

本实现并非纯信息素方案，而是结合了状态机与直接导航：

- **状态机驱动**：蚂蚁根据内部状态（而非仅依赖信息素）决定行为，解决了"做什么"的决策问题
- **直线回巢**：回程蚂蚁直接计算巢穴方向移动，不依赖巢穴信息素导航，避免了自追踪振荡
- **信息素仅用于寻路**：食物信息素仅用于引导外出蚂蚁找到食物，简化了信息素的角色

## 技术细节

- 支持亮色/暗色主题自动切换
- Canvas 高清渲染（适配设备像素比 DPR）
- 支持移动端触摸操作
- 墙壁碰撞检测基于线段相交算法
